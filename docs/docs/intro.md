# Introducción

## Introducción

Este proyecto desarrolla una **plataforma web para la gestión de documentos académicos**, facilitando la interacción entre estudiantes y profesores en procesos de entrega, revisión y descarga de archivos.

- Los **estudiantes** pueden registrarse, iniciar sesión y subir documentos fácilmente.
- Los **profesores** pueden visualizar, descargar y gestionar los archivos desde un panel administrativo.

La plataforma incluye módulos independientes para usuarios, carga y eliminación de archivos, interfaces separadas y carpetas para estilos, scripts y almacenamiento. Además, se integraron **herramientas DevOps** como Jenkins, Docker y Docker Hub, utilizando **tags** para versionamiento y despliegues controlados.

---

## Justificación y Beneficios

El manejo de documentos académicos suele ser complicado usando múltiples canales (correo, chats, plataformas no especializadas), lo que provoca desorganización, pérdida de archivos y dificultades para el seguimiento académico.

Esta plataforma centraliza todo en un sistema intuitivo, mejorando la eficiencia y trazabilidad. La integración de Jenkins, Docker y Docker Hub permite automatizar construcción, despliegue y versionado de manera profesional, introduciendo buenas prácticas de DevOps en un entorno educativo.

---

## Objetivos del Proyecto

**General:** Desarrollar e implementar un sistema web de gestión documental que integre prácticas DevOps para automatizar construcción, versionado y despliegue.

**Específicos:**

- Integrar Jenkins para automatizar pipelines de construcción, test y despliegue.
- Contenerizar la aplicación con Docker para consistencia en distintos entornos.
- Mantener una estructura modular que facilite mantenimiento y escalabilidad.
- Implementar un panel administrativo que permita a los profesores gestionar los documentos de manera eficiente.
